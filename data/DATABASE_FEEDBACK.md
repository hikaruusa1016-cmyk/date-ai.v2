## 🔧 改善が必要な項目

### 1. データの一貫性の問題

#### 1.1 `price_range`の表記揺れ
**現状**:
- `$$` (行4-8)
- `¥¥` (行43, 52, 63)
- `1000-2000` (行2, 89)
- `3000-5000` (行63)
- `無料` (行3)
- `free` (行75, 85)

**推奨**:
```
無料スポット: "0" または "free"
有料スポット: "1000-2000" (数値範囲)
```

**修正例**:
```csv
# 修正前
price_range,budget_level
$$,medium
¥¥,mid

# 修正後
price_range,budget_level
1500-3000,medium
2000-4000,medium
```

---

#### 1.2 `budget_level`の表記揺れ
**現状**:
- `low`, `medium`, `high` (英語)
- `mid` (省略形)
- `中` (日本語)
- `middle` (別の英語表現)

**推奨**: 以下のいずれかに統一
```
low    (低予算: ~3,000円)
medium (中予算: 3,000~8,000円)
high   (高予算: 8,000円~)
```

**修正が必要な行**:
- 行4, 5, 6, 7, 8, 40, 52, 62, 63, 68, 76, 77: `mid` → `medium`
- 行43: `中` → `medium`

---

#### 1.3 `mood_tags`の言語混在
**現状**:
- 日本語: `おしゃれ,カジュアル` (行2)
- 英語: `lively,modern` (行4)
- 混在: `アート, 自然, 散歩` (行3)

**推奨**: 日本語に統一
```csv
# 推奨される mood_tags
"おしゃれ,モダン"
"落ち着いた,大人の雰囲気"
"カジュアル,賑やか"
"ロマンチック,夜景"
"アート的,洗練された"
```

**理由**: ユーザー向け表示が日本語のため

---

#### 1.4 区切り文字の不統一
**現状**:
- `recommended_for`: `first|second|all` (パイプ) と `first,second,all` (カンマ)が混在
- `best_time_slot`: `morning|afternoon` (パイプ)
- `interest_tags`: `cafe,photography` (カンマ)

**推奨**: すべてパイプ`|`に統一（マニュアルに記載の通り）
```csv
# 推奨
recommended_for: "first|second|all"
best_time_slot: "lunch|afternoon|evening"
interest_tags: "gourmet|cafe"
mood_tags: "おしゃれ,モダン"  # これはカンマでOK（表示用テキスト）
```

**現状**: 両方にGoogle Mapsリンクが入っているケースあり

```csv
# 修正後（こちらはOK）

source_url: Google Maps
```
